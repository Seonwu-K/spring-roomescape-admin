### 2. 이번 타임의 학습 전략
- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  - 개념 학습 후 관련 개념이나 실행 과정 등을 흐름으로 묶어서 글로 적어보는 방식을 추가로 할 예정이었고 실제로 실행하였다.
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
 - 기존 2단계 내용과 동일하게 각각의 컨트롤러와 리포지토리를 통해 테이블을 객체로 다루는 방식 까지는 문제가 없었다.
 - 예약과 예약시간 테이블이 조인 하는 기능을 구현하고자 했을 때, 기존 도메인 객체 구조를 어떻게 수정해야 하는지(fk를 상태로 가질 것인지), 미션 요구사항에 맞는 응답 형태로 내보내려면 어떤 방식이 있을지 고민이 되었다.
 - 객체의 구조는 미션 요구사항에 맞춰 따라가기로 하고, 내가 고민했던 fk 방식과의 차이는 리뷰 요청 이후 개인적인 학습 시간을 통해 조사하기로 했다.
 - 그리고 응답 형태 구성의 경우, 가져오는 ResultSet과 RowMapping을 통해 객체 값을 그대로 반환하는 식으로 구성하였다.
 - 이렇게 했을 때, 사용자의 요청으로부터, 어떤 단계를 거쳐서 조인이 이루어지고 그게 원하는 응답값으로 이어지는지 흐름이 제대로 정리가 안된다는 생각이 들어서 내 코드를 직접 따라서 전체 흐름을 한줄의 글로 정리해보았다.
 - 기능 구현에 급급해서 돌아가는 코드를 만들었지만 코드의 흐름을 따라가지 못하고 있었는데, 한번 글로 플로우를 정리했을 때 비로소 전체 구조가 머릿 속으로 정리가 되었다.
