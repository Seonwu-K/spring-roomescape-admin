### 2. 이번 타임의 학습 전략
- 이전에 바꾸기로 한 전략은 무엇이었고, 실행했는가?
  - 학습하고자 하는 내용을 정하고 학습하기 전에, 무엇을 학습하고 학습하지 않을 것인지 정해두고 시작하기
  - 학습 시작 전에 미션에 지장이 가지 않도록 시간 제약을 정해두고 학습하기
- 실제로 어떻게 학습했는지 디테일한 과정을 써보세요.
  - 미션을 진행하던 크루들 사이에서 Repository에 대한 언급이 나왔다
  - 해당 용어에 대한 정확한 정의를 내릴 만큼 이해하고 있지 못했기 때문에, 미션에 적용이 가능한지 여부를 판단할 수가 없었다.
  - Repository의 정확한 정의를 파악하고자 언제 처음 나온 용어인지, 우리가 사용하는 용어는 어떤 도메인에 해당되는 용어인지 정리하였다.
  - 2003년 고안된 DDD 개념을 다루는 책에서 Repository의 정의, 역할을 읽고 Reposiotry 계층이 등장하게 된 이유와 쓰임 용도를 확인하고 미션 1단계에 적용할 이유와 근거가 충분한지 검토하였다. 
  - 우리가 흔히 사용하는 Repository 라는 용어가 자바에서도 스프링 JPA의 Repository에 해당되는 용어 임을 확인하고 스프링 공식 문서에서 정의한 Repository의 용도와 역할을 확인하고 의미를 혼동하지 않도록 정리하였다.
  - 이 과정에서 Repository라는 용어와는 관련 없는, 추가적인 용어들(ex. DDD 책을 학습하면서 DDD란 무엇인가, Repository의 정의에 사용했던 Aggregates는 무엇인가 등 학습이 필요한 부분을 발견해도 메모만 해두고 과감히 스킵하였다)
